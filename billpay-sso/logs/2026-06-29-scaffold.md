# 2026-06-29 — BillPay SSO overview, scaffolded

## What this is
New single-file case study (`index.html`) for a **UI/UX enhancement** of BillDesk
**BillPay SSO** + a **micro-illustration contribution to the DLS** for biller
categories. project-overview skill, reusing the BillDesk `<style>` block (same
family as Netbanking / Wallet / ONDC overviews).

## Research (web)
- Confirmed BillDesk = Indian payments company; BillPay = add-billers / pay-bills
  service reachable via bank net-banking. SSO = bank session carries into the
  portal (no second login). Web did NOT surface a precise bank count or deep SSO
  detail. Used the user-provided "30–50 banks" as the authoritative figure (user
  works at BillDesk). Sources: billdesk.com/web, en.wikipedia.org/wiki/BillDesk,
  bankbazaar bill-payment-through-billdesk.
- Could NOT capture legacy/"before" screenshots from the web — web tooling returns
  markdown text, not images, and scraping product UI isn't appropriate. Told the
  user; built a `assets/before/legacy-add-biller.png` drop-slot instead and framed
  the legacy pain in prose.
