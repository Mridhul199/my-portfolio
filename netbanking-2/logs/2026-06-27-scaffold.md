## What this doc is
A case study of the admin/back-office portal for the Netbanking 2.0 rail. The
portal is operator-facing (bank ops + platform teams), not consumer-facing. It
covers a performance dashboard, user management, bank-statement upload, and
settlements — replacing manual, spreadsheet-driven operations.

## Intake (from the questionnaire)
- **Project:** Admin portal for NPCI/Bharat-BillPay-style "Netbanking 2.0" rail
  (web search confirmed the rail is a new interoperable netbanking payment
  switch; this project is the *operator* console, not the consumer pay flow).
- **Type:** Net-new build — **no existing portal**, so no legacy "before" UI.
  The "before" is the manual process.
- **What was solved:** automated a lot of manual work — settlement
  reconciliation, statement handling, ticket-driven user access; gave operators
  a single performance view.
- **Assets:** screenshots exist; **starting with the dashboard**. Other feature
  screens to be added later.
- **Impact (real, not invented):** 2 banks live in production. Deeper insights /
  UI polish staged for after the user base grows.
